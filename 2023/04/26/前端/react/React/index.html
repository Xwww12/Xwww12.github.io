项目结构

通过React Router创建项目后，会生成以下文件

  • app/：源代码目录，相当于传统的 src/ 文件夹

  • node_modules/ ：存放项目依赖的文件夹，所有下载的依赖都会放这里

  • public/ ：静态资源文件夹，放在这里的文件不会被打包工具（Vite）处理，而是原样拷贝到最终构建目录

  • package.json ：项目的核心配置文件

    包含：

    • 项目名称、版本、作者信息
    • 脚本命令（如 dev、build、preview）
    • 依赖列表
    • 工具配置项（如 eslint、prettier）

  • package-lock.json：自动生成的依赖锁文件。记录安装的每个包的精确版本，确保在任何机器上安装结果一致

  • react-router.config.ts：框架配置文件，允许您自定义 React Router 应用程序的各个方面，如服务器端渲染、目录位置和构建设置

  • tsconfig.json：TypeScript 的配置文件

    • 输出目录
    • 语法特性
    • 路径别名
    • 严格模式等

  • vite.config.ts：Vite 的核心配置文件

    • 配置插件（如 React、TSX 支持）
    • 定义别名、代理、构建目标
    • 自定义服务器端口等

组件渲染

渲染做了什么：

  1. 重新执行函数组件函数
  2. 根据返回的JSX，生成新的虚拟DOM树
  3. 将新虚拟DOM树和旧的比较
  4. 只更新真实DOM的变化的部分

渲染时机：

  • 应用启动时初次渲染
  • 组件或祖先组件状态发生改变时重渲染（修改state会被标记要需要重新渲染）

state

和普通变量的区别：

  • 更改state会触发组件重新渲染，普通变量不会

  • 普通变量在组件重新渲染时不会保存当前值，state会保留渲染之前的值

  • 应将state视为只读

  • 组件不被渲染时，对应的state也会被销毁

  • state是通过组件在渲染树的位置来保存的，比如相同位置且相同的组件，切换时会保留state

    同一个位置指的是像 ? <Button/> : <Button/>、if else这样的，而<> {<Button/>} {<Button/>} </>这种的就不是同一个位置

使用：

注意：

  • 以use开头的函数称为Hook
  • 不要在循环、条件语句里用。useState()是通过给每个组件维护一个数组来保存state，每次渲染都是按顺序把值重新赋给变量
  • 如果是普通变量改变，不会触发重新渲染，新的值不会显示在组件上
  • 修改state会生成新的快照，在下次渲染生效
  • 可以用更新函数index => index + 1，这样会把更新后的值传给下一个函数

更新数组

  • 删除元素: 使用filter, filter会创建一个新数组
  • 修改/替换元素：使用map来遍历并找到要修改的
  • 特定位置插入元素：使用...数组.slice(起始坐标, 终点坐标)展开前面和后面，然后往中间插入

Reducer管理state

将state迁移至Reducer方便管理

优点：

  1. 比较好读懂
  2. 好调试
  3. 后期代码量比state低

步骤：

  1. 定义操作：将设置状态的逻辑修改成dispatch的一个action
  2. 实现操作的具体步骤：编写reducer函数
  3. 使用useReducer代替useState
  4. 还可以用useImmerReducer进一步简化

注意：

  • reducer应该只用来计算下一个状态，而不应该做其他事

ref

和state的区别：更新ref时不会触发重新渲染；更改会立即表现出来，而不是等到下次渲染

和普通变量的区别：每次重新渲染时，raect会记住ref的值

何时使用：很少用到，要存储一些值，但不影响渲染逻辑
