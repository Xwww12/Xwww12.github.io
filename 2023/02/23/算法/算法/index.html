格雷码

在格雷码中，任意两个相邻的代码只有一位二进制数不同，最大码与最小码之间也仅一位不同

转换方式：普通二进制记为Bn-1 Bn-2 … B0，格雷码记为Gn-1 Gn-2 … G0

Gn-1 = Bn-1;

Gi = Bi+1 异或 Bi (0 <= i <= n - 2)

前缀和

设数组nums，前缀和数组s，则s[i + 1] = s[i] + nums[i]，s[i]表示nums[i]前面元素之和

通过前缀和，我们可以把子数组的元素和转换成两个前缀和的差

如nums = [1, 2, -1, 2]，则子数组[2, -1, 2]的和为s[4] - s[1] = 3

排序

计数排序

设数组长度为n，数组最大值和最小值差为k

时间复杂度：O(n + k)

空间复杂度：O(k)

适用范围：k远小于n时，时间复杂度可视为O(n)，优于其他基于比较的排序

快速排序

时间复杂度：最好O(nlogn)，最差O(n ^ 2)

空间复杂度：O(1)

归并排序

时间复杂度：O(nlogn)

空间复杂度：O(n)

同余

​ 定义：两个数x和y，如果(x - y) mod m = 0，则称x与y对模m同余，记作：x = y (mod m)

也就是说两个数模一个数后得到相同的数，例如 42 = 12 （mod 10）、-17 = 3 （mod 10）

​ 处理取模技巧：如果数x为负数，则 x mod m = y mod m 等同于(x mod m + m) mod m = y mod m

例题：6321. 执行操作后的最大 MEX - 力扣（LeetCode）

数位DP

​ 考虑人类计数的方式，最朴素的计数就是从小到大开始依次加一。但我们发现对于位数比较多的数，这样的过程中有许多重复的部分。例如，从 7000 数到 7999、从 8000 数到 8999、和从 9000 数到 9999 的过程非常相似，它们都是后三位从 000 变到 999，不一样的地方只有千位这一位，所以我们可以把这些过程归并起来，将这些过程中产生的计数答案也都存在一个通用的数组里。

特征：

  1. 数据范围非常大(不能暴力)
  2. 和每个数位有关
  3. 有明显的数学风味
  4. 喜欢问范围[L, R]中有多少(不)符合的数
  5. 模板性很强

字符串匹配KMP

如果暴力匹配，如果模式串和主串不匹配时，模式串需要从头开始匹配；

使用KMP的话，可以通过next数组来回退模式串来继续匹配，不用再从头开始

  • next数组：记录了模式串与主串(文本串)不匹配的时候，模式串应该从哪里开始重新匹配

next 计算方式为最长公共前缀+1

这里的next是优化后的,

口算的方式为先计算普通的next数组，

然后遍历next，找对应指向的下标-1的位置的字符是否与当前的相等,

如果相等就将值改为它的值，否则不变

最短路径

Floyd算法

Floyd算法的基本思想是通过逐个考虑每一个节点作为中间节点，更新所有节点之间的最短路径。

Floyd算法属于动态规划

时间复杂度O(n^3)，适合节点少的图

Dijkstra算法

Dijkstra算法的基本思想是从起点开始，依次寻找距离起点最近的节点，并更新其相邻节点的距离和路径。在每一次迭代中，Dijkstra算法会选择当前距离起点最近的未标记节点，将其标记为已访问，并更新其相邻节点的距离和路径

Dijkstra算法属于贪心
