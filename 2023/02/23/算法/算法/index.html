      设从起点到环形入口的距离为x，从起点到相遇点的距离为y，从相遇点到入口的距离为z;

      则慢指针走的距离为 x + y，快指针走的距离为x + y + n * (y + z);

      而快指针的速度是慢指针的两倍，因此 2 * (x + y) = x + y + n * (y + z) (n >= 1);

      整理得 x = (n - 1) * (y + z) + z;
      意味着让两个新的指针，一个从链表起点出发，一个从相遇点出发，会在第二个指针绕(n - 1)圈后再入口和第一个指针相遇

在格雷码中，任意两个相邻的代码只有一位二进制数不同，最大码与最小码之间也仅一位不同

转换方式：普通二进制记为Bn-1 Bn-2 … B0，格雷码记为Gn-1 Gn-2 … G0

Gn-1 = Bn-1;

Gi = Bi+1 异或 Bi (0 <= i <= n - 2)

同余

​ 定义：两个数x和y，如果(x - y) mod m = 0，则称x与y对模m同余，记作：x = y (mod m)

也就是说两个数模一个数后得到相同的数，例如 42 = 12 （mod 10）、-17 = 3 （mod 10）

​ 处理取模技巧：如果数x为负数，则 x mod m = y mod m 等同于(x mod m + m) mod m = y mod m

例题：6321. 执行操作后的最大 MEX - 力扣（LeetCode）

背包问题

01背包

  • 特征：每个物品只能拿一次，求能拿的最大价值

  • 做法：因为不能重复选同一个物品，所以要从大到小遍历容器， 避免同一个物品被多次计算

    1. 定义dp[j] = 背包容量为 j 时的最大价值
    2. 遍历物品+容器

  • 题：

    1. 01背包+记忆化搜索：494. 目标和
    2. 二维01背包：474. 一和零
    3. 难点在于发现是01背包问题：1049. 最后一块石头的重量 II

完全背包

  • 特征：每个物品的数量是无限的
  • 做法：从小到大遍历容器，让物品可以被重复选择
    1. 定义dp[j] = 背包容量为 j 时的最大价值
    2. 遍历物品+容器
  • 题：
    1. 322. 零钱兑换
    2. CSP202503B. 机器人饲养指南
    3. 279. 完全平方数
    4. 稍有不同的完全背包：139. 单词拆分

多重背包

  • 特征：每个物品的数量有限个

  • 做法：当做01背包做，要从大到小遍历容器

    1. 定义dp[j] = 背包容量为 j 时的最大价值
    2. 遍历物品+容器+选的物品个数

数位DP

​ 考虑人类计数的方式，最朴素的计数就是从小到大开始依次加一。但我们发现对于位数比较多的数，这样的过程中有许多重复的部分。例如，从 7000 数到 7999、从 8000 数到 8999、和从 9000 数到 9999 的过程非常相似，它们都是后三位从 000 变到 999，不一样的地方只有千位这一位，所以我们可以把这些过程归并起来，将这些过程中产生的计数答案也都存在一个通用的数组里。

特征：

  1. 数据范围非常大(不能暴力)
  2. 和每个数位有关
  3. 有明显的数学风味
  4. 喜欢问范围[L, R]中有多少(不)符合的数
  5. 模板性很强

题集：

  • 1012. 至少有 1 位重复的数字 - 力扣（LeetCode）
  • 233. 数字 1 的个数

最短路径

Floyd算法

Floyd算法的基本思想是通过逐个考虑每一个节点作为中间节点，更新所有节点之间的最短路径。

Floyd算法属于动态规划

时间复杂度O(n^3)，适合节点少的图

Dijkstra算法

Dijkstra算法的基本思想是从起点开始，依次寻找距离起点最近的节点，并更新其相邻节点的距离和路径。在每一次迭代中，Dijkstra算法会选择当前距离起点最近的未标记节点，将其标记为已访问，并更新其相邻节点的距离和路径

Dijkstra算法属于贪心
